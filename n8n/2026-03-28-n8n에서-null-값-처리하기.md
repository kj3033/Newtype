### Context
n8n 워크플로우를 설계할 때 이전 노드에서 값이 전달되지 않거나 API 응답에서 특정 필드가 누락되어 `null` 또는 `undefined`가 발생하는 경우가 빈번합니다. 이러한 값을 적절히 처리하지 않으면 후속 노드에서 에러가 발생하거나 데이터 흐름이 중단될 수 있습니다. 

### Core
n8n의 표현식(Expressions)에서 `null` 또는 `undefined`를 처리하는 가장 효과적인 방법은 자바스크립트의 논리 연산자(Logical Operators)를 사용하는 것입니다.

* 논리합(OR) 연산자(`||`)를 사용한 기본값 할당
```javascript
{{ $json.field || '기본값' }}
```

* 삼항 연산자(Ternary Operator)를 사용한 조건부 처리
```javascript
{{ $json.field ? $json.field : '값이 없음' }}
```

* 옵셔널 체이닝(Optional Chaining)을 사용한 안전한 접근
```javascript
{{ $json.data?.user?.name || '익명' }}
```

### Insight
`null` 값 처리는 워크플로우의 안정성(Robustness)을 결정짓는 핵심 요소입니다. 특히 외부 API를 연동할 때는 응답값이 비어있을 가능성을 항상 염두에 두어야 합니다. 논리합(`||`) 연산자는 가장 간결하게 기본값을 설정할 수 있어 가독성이 높지만, 입력값이 `0`이나 `false`와 같은 Falsy 값일 때도 기본값으로 대체될 수 있다는 점을 주의해야 합니다. 데이터의 타입이 엄격하게 구분되어야 하는 환경이라면 삼항 연산자를 사용하여 명시적으로 검사하는 것을 권장합니다.
