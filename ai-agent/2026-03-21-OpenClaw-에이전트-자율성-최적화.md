### Context
OpenClaw 보안 업데이트 이후 에이전트의 시스템 명령어 실행 시 매번 승인 팝업이 발생하거나 권한 인증을 요구하는 환경이 조성되었습니다. 이는 에이전트의 자율적 작업 흐름을 방해하므로, 특정 환경에서 에이전트의 실행 권한을 최적화하고 보안 정책을 조정하여 자동화 효율성을 높일 필요가 있습니다.

### Core
에이전트의 실행 자율성을 확보하기 위해 다음 두 가지 설정을 적용합니다.

* **Exec 도구 자동 승인 설정**
에이전트가 실행하는 명령어가 사용자 승인 없이 즉시 수행되도록 설정합니다.

```bash
# 명령어 기반 설정
openclaw config set tools.exec.security full
openclaw config set tools.exec.ask off

# 또는 ~/.openclaw/config.json 수정
{
  "tools": {
    "exec": {
      "security": "full",
      "ask": "off"
    }
  }
}
```

* **Sudoers 패스워드 무암호화(NOPASSWD)**
루트(Root) 권한이 필요한 시스템 작업(apt, systemctl 등)을 에이전트가 중단 없이 수행하도록 설정합니다.

```bash
# 설정 파일 생성
echo 'jslee ALL=(ALL) NOPASSWD: ALL' | sudo tee /etc/sudoers.d/jslee-nopasswd
sudo chmod 440 /etc/sudoers.d/jslee-nopasswd
```

* **임시 권한 상향**
전역 설정을 변경하지 않고 특정 세션에서만 권한을 상향하려면 에이전트에게 다음 지시어를 전달합니다.

```text
/elevated full
```

### Insight
에이전트의 자율성을 극대화하기 위해 `exec` 승인 비활성화와 `NOPASSWD` 설정을 조합하는 것은 매우 강력하지만, 보안 위험(Security Risk)이 수반됩니다. 모든 명령이 승인 없이 루트 권한으로 실행될 수 있으므로, 해당 에이전트가 접근 가능한 파일 시스템 범위와 네트워크를 격리된 환경(Containerized Environment) 내에서 운용하는 것을 권장합니다. 프로덕션 환경에서는 반드시 특정 작업에 필요한 최소 권한(Least Privilege)만 부여하는 방향으로 정책을 재검토해야 합니다.
